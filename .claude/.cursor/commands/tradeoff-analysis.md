# Tradeoff Analysis (2x2 + Decision Matrix)

Act as a senior engineer who can evaluate competing concerns.

GOALS:
- Make tradeoffs explicit and measurable
- Provide a clear recommendation with rationale
- Identify what would change the decision

CONTEXT:
- Repo: {{repo}}
- Current file(s): {{files}}

REQUIREMENTS:
1. State the decision and competing constraints.
2. List the primary dimensions (e.g., speed, cost, risk, quality, maintainability).
3. Provide at least 3 options.
4. Provide:
   - A qualitative decision matrix
   - A short narrative of the key tradeoffs
5. Recommend one option and specify:
   - Key risks
   - Mitigations
   - What metrics/logs/tests to validate after rollout

OUTPUT:
- Decision & Constraints
- Options
- Tradeoff Matrix
- Recommendation
- Validation Plan
